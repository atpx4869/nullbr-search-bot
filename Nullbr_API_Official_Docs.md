# NULLBR 影视资源 API 官方文档总结

本文档基于官方 [API 文档页面 (nullbr.com/api)](https://nullbr.com/api) 整理，涵盖了目前提供的所有影视元数据及资源接口信息。此文档旨在方便本地开发查询，避免频繁访问网页。

---

## 1. 基础信息与认证 (Authentication)

- **API 基础 URL**: `https://api.nullbr.eu.org`
- **请求方法**: 绝大多数接口使用 `GET`（获取用户信息配置时可能会使用 `POST`）。
- **返回格式**: 标准 JSON。

### 认证方式 (Headers)
API 调用依赖两种 Header 凭证：
1. `X-APP-ID`: **必须包含在所有请求中**。每个注册用户最多可以申请 3 个 APP ID（在“应用中心”获取）。
2. `X-API-KEY`: **获取具体资源 (RES) 时必须携带**。调用此类接口会消耗用户的请求配额。

### 接口分类 (Category)
- **META** (元数据类): 仅需 `X-APP-ID`，不消耗配额。包含影视的详情、简介和海报等。
- **RES** (资源类): 需要 `X-APP-ID` 和 `X-API-KEY`。包含 115 链接、磁力链接、ed2k 和 m3u8 等，消耗配额。
- **USER** (用户类): 账号配额与订阅管理接口。

## 3. 开发规范与官方寄语 (Tips & FAQ)

1. **响应格式**: 全面上云规范 JSON 格式，包含标准的 `tmdbid` 对应信息，资源返回多分辨率及字幕等提示。
2. **应用接入建议**: 
   - 官方鼓励开发小而美的检索工具（如机器人/浏览器插件），或者自动化媒体库（Jellyfin/Emby/Plex）补全服务。
   - 不推荐利用该 API 快速批量建立同质化的资源分享网站。
3. **API 限制与更新**:
   - 如果频率过高触发 HTTP 429 会有冷却惩罚。
   - 后台资源保持每日一更（每日增量幅度在 0.5% ~ 1%）。
4. **NSFW 板块**: 现阶段 API 暂未提供该数据分类，未来 1-2 个月内支持 DMM 数据对齐后有望开放。
